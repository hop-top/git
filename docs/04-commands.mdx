# Commands

git-hop provides a set of commands for managing hubs, hopspaces, and worktrees.

## git hop `&lt;uri&gt;`

Create a new hub and hopspace for a remote repository.

### Usage

```bash
git hop &lt;uri&gt; [flags]
```

### Examples

```bash tab="HTTPS"
git hop https://github.com/org/repo.git
```

```bash tab="SSH"
git hop git@github.com:org/repo.git
```

```bash tab="File Protocol"
git hop file:///path/to/local/repo
```

### What Happens

1. Detects server and organization
2. Creates hub directory in current working directory
3. Creates hopspace under `$GIT_HOP_DATA_HOME/org/repo`
4. Clones repo into hopspace (default branch only)
5. Symlinks worktree back into hub
6. Auto-starts environment if services defined

### Output

```
Creating hub for org/repo
Created hopspace for 'main'
Worktree: ./org/repo/main
Ports: 10100-10102
Services: api, db
```

## git hop `&lt;branch&gt;`

Inside a hub: create a worktree for a branch.

### Usage

```bash
git hop &lt;branch&gt; [flags]
```

### Examples

```bash
git hop feature-x
```

```bash tab="From Remote"
git hop origin/pr-123
```

```bash tab="Local Branch"
git hop my-local-branch
```

### What Happens

1. Validates current directory is a hub
2. Creates hopspace (if missing) under `$GIT_HOP_DATA_HOME/org/repo`
3. Creates worktree for specified branch
4. Generates deterministic ports and volumes
5. Auto-starts environment (if configured)

### Output

```
Created hopspace for 'feature-x'
Worktree: ./feature-x
Ports: 11500-11505
Services: api, db, cache
```

## git hop status

Show detailed environment state for current worktree or hub.

### Usage

```bash
git hop status [flags]
```

### Examples

```bash
git hop status
```

### Output (worktree)

```
On branch feature-x
nothing to commit, working tree clean

Services:
  api    running  http://localhost:11500
  db     running  postgresql://localhost:11501
  cache  running  redis://localhost:11502
```

### Output (hub)

When run from a hub, the table includes a `Status` column reporting
each worktree's position relative to the hub's default branch.

```
Hub: org/repo
Location: /Users/jadb/.w/git
╭──────────────┬─────────┬───────────────────────────────┬──────────────╮
│ BRANCH       │ STATE   │ STATUS                        │ PATH         │
├──────────────┼─────────┼───────────────────────────────┼──────────────┤
│ main         │ Linked  │ default                       │ hops/main    │
│ feat/clean   │ Linked  │ synced                        │ hops/clean   │
│ feat/done    │ Linked  │ merged (4 behind)             │ hops/done    │
│ feat/wip     │ Linked  │ 3 ahead                       │ hops/wip     │
│ feat/old     │ Linked  │ diverged (1 ahead, 12 behind) │ hops/old     │
│ feat/gone    │ Missing │ -                             │ hops/gone    │
╰──────────────┴─────────┴───────────────────────────────┴──────────────╯
```

Status values:

| Value | Meaning |
|-------|---------|
| `default` | This is the hub's default branch |
| `synced` | Same tip as default — no commits ahead or behind |
| `merged (N behind)` | Branch tip is reachable from default; default has moved on by N commits |
| `N ahead` | Branch has N commits not yet on default |
| `diverged (N ahead, M behind)` | Branch and default have both moved |
| `unknown` | Couldn't compute (e.g. default branch ref missing locally) |
| `-` | Worktree directory is missing on disk |

## git hop env

Control environment services.

### Usage

```bash
git hop env &lt;subcommand&gt; [flags]
```

### Subcommands

#### start

Start services for current worktree.

```bash
git hop env start
```

Output:

```
Starting environment for branch: feature-x
  ✓ api started on port 11500
  ✓ db started on port 11501
  ✓ cache started on port 11502
```

#### stop

Stop services for current worktree.

```bash
git hop env stop
```

Output:

```
Stopping environment...
  ✓ api stopped
  ✓ db stopped
  ✓ cache stopped
```

#### restart

Restart services for current worktree.

```bash
git hop env restart
```

#### gc

Garbage collect orphaned dependencies.

```bash
git hop env gc [flags]
```

**Flags:**
- `--dry-run` - Preview what will be deleted without actually deleting
- `--force` - Skip confirmation prompt

**What it does:**
- Scans all worktrees to identify orphaned dependencies
- Shows dependency info including last used time and size
- Prompts for confirmation before deletion (unless `--force`)
- Reports total space reclaimed

**Output:**

```
Running dependency audit...
  ✓ Scanned 3 worktrees
  ✓ Updated dependency registry

Orphaned dependencies:
  node_modules.def456  (last used: 7 days ago)  ~120MB
  venv.jkl012         (last used: 2 days ago)   ~45MB

Total reclaimable: 165MB

Delete these dependencies? [y/N]: y
  ✓ Deleted 2 orphaned dependencies
  ✓ Reclaimed 165MB
```

## git hop remove

Remove a hub, hopspace, or branch.

### Usage

```bash
git hop remove &lt;target&gt; [flags]
```

### Examples

```bash tab="Remove Branch"
git hop remove feature-x
```

```bash tab="Remove Hub"
cd ..
git hop remove my-project
```

```bash tab="Unmerged Branch"
git hop remove feature-x --force
```

```bash tab="Dirty Worktree"
git hop remove feature-x --no-verify
```

### Safety Gate

`remove` probes the worktree before deleting and requires explicit
flags when the operation would lose work. A clean, merged branch
removes silently — no prompt, no warning.

| Branch state | Required flags |
|--------------|----------------|
| Merged into default, clean | (none — silent pass) |
| Merged into default, dirty (uncommitted or untracked) | `--no-verify` |
| Not merged, but pushed to `origin/<branch>` | `--force` |
| Not merged and not pushed | `--force --no-verify` |

When the gate is not satisfied, `remove` exits with status 1 and
prints which flag(s) to add. The gate cannot be bypassed any other
way.

### `--force` vs `--no-verify` vs `--no-prompt`

These three flags do **different** things. They are not interchangeable.

| Flag | Purpose | Bypasses gate? |
|------|---------|----------------|
| `--force` | Allow removing a branch that is **not merged** into the default branch | Yes — for the unmerged case |
| `--no-verify` | Allow removing a worktree with **uncommitted/untracked files** or **unpushed commits** | Yes — for the dirty/unpushed case |
| `--no-prompt` | Skip the post-gate "Really delete?" confirmation. Useful in scripts | **No** — never bypasses the gate |

In particular, `--no-prompt` alone on a risky removal will **fail
fast with exit 1**, not silently delete. You must combine it with
the appropriate `--force` / `--no-verify` to actually remove. Example:

```bash
# Branch is unmerged + dirty + unpushed, automated cleanup:
git hop remove feature-x --force --no-verify --no-prompt

# Just --no-prompt on the same branch → exit 1 with gate error.
git hop remove feature-x --no-prompt   # ✗ refuses
```

For hub removal, only `--no-prompt` matters (there is no gate; the
prompt is the only confirmation).

### What Gets Removed

- Worktree
- Hopspace data
- Port and volume allocations
- Containers
- Networks

## git hop prune

Remove orphaned or broken hopspaces.

### Usage

```bash
git hop prune [flags]
```

### Examples

```bash
git hop prune
```

```bash tab="Dry Run"
git hop prune --dry-run
```

### What It Does

- Detects worktrees without corresponding Git refs
- Identifies containers without hopspaces
- Finds orphaned volumes
- Cleans up broken symlinks

## git hop doctor

Check for inconsistencies and print diagnostics. Use `--fix` to automatically repair issues.

### Usage

```bash
git hop doctor [flags]
```

### Examples

```bash
git hop doctor
```

```bash tab="Auto-fix Issues"
git hop doctor --fix
```

### Flags

- `--fix` - Automatically repair detected issues

### What It Checks

- Path configuration (data, config, cache directories)
- Hub configuration and symlinks
- Hopspace existence and consistency
- Orphaned worktrees
- Dependency health (symlinks, shared storage, orphaned deps)

### Output

```
Running git-hop diagnostics...

=== Checking Paths ===
Data home:   /home/user/.local/share/git-hop
Config home: /home/user/.config
Cache home:  /home/user/.cache

=== Checking Hub ===
Hub found at: /projects/my-repo
Expected hopspace: /home/user/.local/share/git-hop/org/repo
✓ Hopspace exists

=== Checking Dependencies ===
Dependencies Status:
  ✓ node_modules.abc123 used by: main, feature-x
  ✓ vendor.789ghi used by: main
  ⚠ venv.jkl012 orphaned (no branches use it) - run 'git hop env gc' to clean
  ✗ feature-y: broken symlink node_modules → (missing abc999)
  ⚠ feature-z: has local node_modules (720MB) instead of symlink

Recommendations:
  - Run 'git hop doctor --fix' to restore shared deps
  - Run 'git hop env gc' to reclaim 45MB from orphaned deps

=== Summary ===
Issues found. Run 'git hop doctor --fix' to automatically repair them.
```

### Auto-fix

When issues are found, use `--fix` to automatically repair them:

```bash
git hop doctor --fix
```

The `--fix` flag will:
- Create missing directories
- Initialize missing hopspaces
- Register branches from hub to hopspace
- Sync hub-hopspace configurations

### Flags

| Flag | Description |
|------|-------------|
| `--fix` | Automatically fix issues |

## Command Aliases

Many commands have shorter aliases for convenience:

| Command | Aliases | Description |
|---------|---------|-------------|
| `list` | `ls`, `all` | List hubs, hopspaces, and branches |
| `add` | `create`, `new` | Add a new worktree and environment |
| `remove` | `rm`, `delete`, `del` | Remove a hub, hopspace, or branch |
| `status` | `st`, `info` | Show the working tree status |
| `prune` | `cleanup`, `clean` | Remove orphaned data |
| `doctor` | `check`, `repair` | Check and repair the environment |
| `init` | `setup`, `install` | Initialize git-hop repository structure |
| `env start` | `env up` | Start environment services |
| `env stop` | `env down` | Stop environment services |

### Examples

```bash
# These are equivalent
git hop list
git hop ls
git hop all

# These are equivalent
git hop add feature-1
git hop create feature-1
git hop new feature-1

# These are equivalent
git hop remove old-branch
git hop rm old-branch
git hop delete old-branch

# These are equivalent
git hop status
git hop st
git hop info

# These are equivalent
git hop env start
git hop env up
```

## Global Flags

These flags work with all commands:

| Flag | Description |
|------|-------------|
| `--json` | Output in JSON format (shorthand for `--format json`) |
| `--format <format>` | Output format: `table`, `json`, `porcelain`, `raw` |
| `--quiet, -q` | Suppress output |
| `--verbose, -v` | Enable verbose output |
| `--force` | Force operation (use with caution) |
| `--hub &lt;name|path&gt;` | Specify hub name or path |
| `--hopspace &lt;name|path&gt;` | Specify hopspace name or path |
| `--dry-run` | Show what would happen without making changes |
| `--help, -h` | Show help for command |
| `--version` | Show git-hop version |
